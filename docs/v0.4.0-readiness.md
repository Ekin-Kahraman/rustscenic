# v0.4.0 readiness note

State as of v0.3.11 (sha c580efb), assessed against `docs/release-gate.md`.

## What is proven

10 of 11 publication-threshold gate items are satisfied (see release-gate § 4 for the canonical list). Headline evidence:

- **Real-data full SCENIC+ end-to-end via the public `pipeline.run`** on two independent 10x multiome datasets:
  - PBMC 3k (human, adult, immune): `validation/multiome_pipeline_run_v0.3.9.json` — 1,091 eRegulons, 451 s wall, 3.67 GB peak RSS, 4/4 canonical PBMC lineage TFs.
  - Mouse brain E18 5k (mouse, embryonic, CNS): `validation/multiome_pipeline_run_v0.3.10_brain_e18.json` — 1,125 eRegulons, 826 s wall, 4.01 GB peak RSS, **9/9 expected E18 cortex marker TFs** recovered as regulons.
- **GRN parity vs current pyscenic 0.12.1 + arboreto 0.1.6** on identical PBMC 3k fixture: per-edge Spearman 0.611 on 480,680 shared edges, within-TF Spearman mean 0.632, 1.78× wall speedup. `validation/parity_v0310/grn_parity_pbmc3k_full.json`.
- **Bit-identical determinism** under same seed (live + inline Rust tests).
- **Fresh-install matrix CI** runs each `[extra]` install path on every tag push (`audit.yml` install-matrix job).
- **GRN truncation knobs + small-n warning** added in v0.3.11 for under-determined inputs (Fuaad's Kamath case).
- **Auditable evidence schema** for every claim: dataset, command, version/SHA, hardware, time, memory, outputs, biology checks, caveats. Documented in `~/.claude/projects/-Users-ekin/memory/project_rustscenic.md`.

## What remains caveated

1. **Raw 10x `pipeline.run` without caller-side ATAC pre-subset** (the one unchecked gate item). The v0.3.7 attempt wedged at GRN for >3 h with topics running over the unsubsetted 451 k-barcode matrix. Documented workaround: caller subsets ATAC to RNA-QC'd cells and passes `adata_atac=...` (the v0.3.9 path). Both real-data smokes use this path; both succeed.
2. **Cistarget kernel parity vs ctxcore** is listed in release-gate § 3 but the v0.3.10 parity refresh covered only GRN. Region-cistarget-kernel parity remains an open sub-item; not gate-blocking but worth a v0.4.x follow-up.
3. **AUCell wall-time and per-cell Pearson logs** are pre-existing (validation/aucell_*.py) and not regenerated per release. Numbers in README are still from 2026-04. Worth refreshing under the `[reference]` Docker image to track upstream pyscenic drift.
4. **Cell-level Kamath arboreto comparison** OOM'd in the 6 GB colima VM. rustscenic cell-level run completed (49 min, 12/12 canonical DA TFs) but the cross-pipeline cell-level number for that dataset is missing. Audit doc explicitly flags this.

None of (1)-(4) prevent users from running rustscenic correctly today via the documented workflow.

## Should raw-10x without caller-side ATAC subset block v0.4.0?

**No. Move it to v0.5.**

Reasoning:
- The pre-subset `adata_atac` path is the *documented* workflow in `docs/tester-quickstart.md` and `docs/release-gate.md`. Users following the docs already do the right thing.
- Two independent real-data smokes (different species, tissue, developmental stage) prove that path works end-to-end with non-empty eRegulons and biology recovery.
- The unchecked item is a *convenience* gap (orchestrator does the subset itself instead of asking the caller), not a *correctness* gap. No published claim depends on it.
- Fixing it requires either (a) in-orchestrator subset based on RNA QC obs, or (b) fragments-side prefilter. Both are real engineering, not a one-line fix. v0.5 sized work.
- The "publishable" label is about whether users can produce reproducible correct results. They can.

The release-gate doc text already calls this "a documented workflow caveat, not a correctness gap" (§ 4 footer). v0.4.0 should ship with the same wording in its release notes.

## Workflow

For real 10x multiome, the documented path is:

1. Load RNA, run cell QC.
2. Build the ATAC peak matrix from fragments + peaks BED.
3. Subset both to RNA-QC'd cells (this is essential; see caveat below).
4. Pass both to `rustscenic.pipeline.run(rna=..., adata_atac=..., ...)`.

See `docs/tester-quickstart.md` for the canonical script.

## Known caveat

`pipeline.run` does not yet pre-subset ATAC fragments based on RNA QC
inside the orchestrator. The caller must do this and pass `adata_atac=...`
(see step 3 above). Without it, an unsubsetted ~450 k-barcode 10x
fragments file will stall topics + GRN. This is a workflow caveat, not a
correctness gap; both real-data smokes in this release use the
caller-side-subset path. Closing the gap inside the orchestrator is
tracked for v0.5.

## What's not regenerated for this release

- AUCell wall-time + per-cell Pearson numbers in README: still from the
  2026-04 logs. Refresh planned for a v0.4.x point release.
- Region-cistarget kernel parity vs ctxcore: separate from the v0.3.10 GRN
  parity refresh; tracked for v0.4.x.
